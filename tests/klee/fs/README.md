# 文件系统安全属性测试

## 概述

本测试使用KLEE符号执行工具对文件系统模块进行安全属性验证。测试主要关注以下几个关键安全属性：

1. **文件操作边界检查**
   - 路径长度限制
   - 缓冲区溢出防护
   - 文件描述符范围检查

2. **内存访问安全**
   - 指针有效性验证
   - 内存边界检查
   - 空指针防护

3. **路径规范化安全**
   - 路径注入防护
   - 目录遍历防护
   - 符号链接处理

4. **文件描述符管理**
   - 描述符分配策略
   - 资源泄漏防护
   - 并发访问控制

5. **加密操作安全**
   - 密钥管理
   - 加密算法实现
   - 数据完整性验证

## 测试用例

### 1. 路径规范化测试 (test_fs_fnNormalize)
- 验证路径规范化函数的安全性
- 检查路径长度限制
- 确保路径格式正确

### 2. 文件打开测试 (test_fs_open)
- 验证文件打开操作的安全性
- 检查文件描述符分配
- 测试不同打开模式

### 3. 文件读写测试 (test_fs_read_write)
- 验证读写操作的安全性
- 检查缓冲区管理
- 测试数据完整性

### 4. 文件系统初始化测试 (test_fs_ini)
- 验证初始化过程的安全性
- 检查加密密钥处理
- 测试系统状态验证

## 使用方法

### 环境要求
- LLVM/Clang 编译器
- KLEE 符号执行工具
- uClibc 库

### 编译和运行
```bash
# 编译并运行所有测试
make

# 清理生成的文件
make clean

# 显示帮助信息
make help
```

### 查看结果
测试完成后，结果将保存在 `klee-out-*` 目录中。可以使用以下命令查看具体信息：

```bash
# 查看测试覆盖率
klee-stats klee-out-*

# 查看具体错误路径
klee-replay ./klee_fs_test klee-out-*/test000001.ktest
```

## 注意事项

1. **符号执行限制**
   - 路径爆炸问题
   - 外部函数调用
   - 浮点运算处理

2. **测试覆盖范围**
   - 仅覆盖核心安全属性
   - 可能需要补充测试用例
   - 考虑边界条件

3. **性能考虑**
   - 测试时间可能较长
   - 内存使用量较大
   - 需要适当配置资源
